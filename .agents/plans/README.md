# Plans

This directory stores task plans created before implementation work begins.

## Naming

Use:

`YYYY-MM-DD-short-task-name.md`

## Required contents

Each plan must include at least:
- a clear objective
- a checklist of tasks
- a detailed step-by-step implementation plan
- risks/questions
- validation notes

## Process

1. Create or update the relevant plan before making non-trivial changes.
2. Keep the checklist updated during execution.
3. Record notable decisions and deviations from the original plan.
4. Leave the final plan file reflecting what was actually done.
